# Cub3D

## 개요

- `cub3D`는 2D 격자 맵을 1인칭 시점의 3D 공간으로 렌더링하는 **raycasting** 그래픽 과제다.
- 핵심은 `.cub` 설정 파일을 검증해 맵과 리소스를 준비하고, 플레이어 위치에서 화면의 각 열로 광선을 쏘아 벽까지의 거리를 계산한 뒤 MiniLibX 창에 텍스처와 색상을 그리는 것이다.

## 과제 요구사항

### Mandatory

- 프로그램은 `.cub` 확장자의 맵 파일 하나를 인자로 받아 실행한다.
- MiniLibX를 사용해 창을 생성하고 이미지를 출력한다.
- `.cub` 파일에서 `NO`, `SO`, `WE`, `EA` 방향별 텍스처 경로를 읽는다.
- `.cub` 파일에서 `F`, `C` 바닥/천장 RGB 색상을 읽고, 각 색상 값은 `0~255` 범위여야 한다.
- 맵은 `0`, `1`, 공백, 시작 방향 문자 `N`, `S`, `E`, `W`로 구성되며, 플레이어 시작 위치는 하나만 있어야 한다.
- 맵은 벽으로 둘러싸여 있어야 하며, 열린 공간이나 잘못된 문자가 있으면 에러로 처리한다.
- raycasting 방식으로 1인칭 시점 화면을 렌더링하고, 벽 방향에 따라 서로 다른 텍스처를 사용할 수 있어야 한다.
- `W`, `A`, `S`, `D`로 이동하고, 좌우 방향키로 시점을 회전한다.
- `ESC` 키와 창 닫기 버튼은 프로그램을 정상 종료해야 한다.
- 잘못된 설정이나 맵 입력이 들어오면 `Error`로 시작하는 메시지를 출력하고 종료한다.

## 개념 정리

### Raycasting

- **raycasting**은 1인칭 시점의 플레이어 위치에서 여러 방향으로 광선을 쏘고, 각 광선이 처음 만나는 벽이나 물체를 기준으로 화면을 구성하는 렌더링 방식이다.
- `cub3D`는 실제 3D 공간을 모델링하는 것이 아니라, 2D 맵에서 계산한 거리와 방향 정보를 이용해 3D처럼 보이는 화면을 만든다.
- 핵심은 플레이어가 바라보는 시야 안에서 화면의 각 세로줄이 어떤 벽을 보고 있는지 계산하는 것이다.

- 화면의 각 x좌표는 하나의 광선에 대응하며, 플레이어의 위치와 시야 방향을 기준으로 각 광선의 방향을 계산한다.
- 각 광선은 2D 맵의 격자를 따라 진행하면서 처음 만나는 벽을 찾고, 이때 벽까지의 거리를 계산한다.
- 벽까지의 거리가 가까우면 해당 화면 열에 그려지는 벽의 높이가 커지고, 멀면 작아져 원근감이 생긴다.
- 충돌한 벽의 방향과 충돌 위치는 어떤 텍스처를 선택하고, 텍스처의 어느 부분을 화면에 그릴지 결정하는 기준이 된다.

### DDA

- DDA는 격자 공간에서 광선이 다음 x축 경계와 y축 경계 중 어디에 먼저 닿는지 비교하며 한 칸씩 진행하는 알고리즘이다.
- 맵이 2차원 배열로 표현되어 있으므로, 광선을 작은 단위로 계속 이동시키는 대신 격자 경계 단위로 검사하면 벽 충돌 지점을 효율적으로 찾을 수 있다.
- 이 방식은 raycasting에서 벽까지의 거리와 충돌 방향을 얻는 핵심 계산으로 사용된다.

### Camera Vector와 Plane

- 플레이어의 시점은 위치 벡터, 방향 벡터, 카메라 평면 벡터로 표현할 수 있다.
- 방향 벡터는 플레이어가 바라보는 방향을 나타내고, plane 벡터는 화면 너비 방향으로 광선을 펼치는 기준이 된다.
- 시작 방향이 `N`, `S`, `E`, `W` 중 무엇인지에 따라 방향 벡터와 plane 벡터의 초기값이 달라진다.

### Texture Mapping

- texture mapping은 벽과 충돌한 위치를 기준으로 텍스처 이미지의 어느 x좌표를 사용할지 정하고, 화면에 그릴 벽 높이에 맞춰 y좌표를 샘플링하는 과정이다.
- 같은 벽이라도 광선이 어느 방향의 면에 닿았는지에 따라 사용할 텍스처가 달라진다.
- 텍스처 좌표 계산이 어긋나면 벽 방향이 바뀌거나 이미지가 뒤집혀 보일 수 있다.

### MiniLibX와 Event Loop

- MiniLibX는 창 생성, 이미지 생성, 키 입력, 화면 출력 같은 그래픽 기능을 제공하는 라이브러리다.
- 그래픽 프로그램은 한 번 출력하고 끝나는 것이 아니라, loop 안에서 입력을 받고 상태를 갱신한 뒤 다시 화면을 그린다.
- 이 과제에서는 키 입력으로 위치와 시점을 바꾸고, 변경된 상태를 다음 렌더링 프레임에 반영한다.

## 문제 해결의 핵심

### `.cub` 파일 검증

- `.cub` 파일은 텍스처 경로, RGB 색상, 맵 데이터가 섞여 있으므로 설정 정보와 맵 영역을 분리해 읽어야 한다.
- 설정 요소가 누락되거나 중복되고, RGB 값이 범위를 벗어나거나, 맵에 허용되지 않는 문자가 있으면 실행 전에 에러로 처리해야 한다.
- 맵은 플레이어가 이동하거나 raycasting이 배열 밖으로 나가지 않도록 외벽으로 닫혀 있어야 한다.

### 좌표계 변환

- 맵 파일은 문자 기반의 2D 배열이지만, 플레이어 위치와 광선 방향은 실수 좌표로 계산한다.
- 플레이어 시작 위치를 타일 중앙으로 보정해야 시작하자마자 벽 경계에 붙어 잘못된 시야가 나오는 상황을 줄일 수 있다.
- 이동할 때는 실수 좌표를 갱신하되, 충돌 검사는 정수 배열 인덱스로 변환해 확인해야 한다.

### 거리 보정과 화면 높이

- 광선이 벽에 닿은 단순 거리를 그대로 쓰면 화면 가장자리에서 물고기눈 왜곡이 생길 수 있다.
- 화면에 그릴 벽 높이는 카메라 평면에 수직인 보정 거리로 계산해야 한다.
- 각 열마다 계산된 거리와 화면 높이를 텍스처 y좌표 비율로 연결해야 벽 이미지가 자연스럽게 늘어나거나 줄어든다.

### 텍스처 선택과 픽셀 출력

- 광선이 x축 방향 벽에 닿았는지, y축 방향 벽에 닿았는지와 광선 방향 부호를 함께 사용해 어떤 텍스처를 그릴지 결정해야 한다.
- 벽의 충돌 지점에서 텍스처 x좌표를 계산하고, 화면 세로 높이에 맞춰 텍스처 y좌표를 순서대로 샘플링한다.
- 바닥과 천장은 별도 텍스처가 아니라 RGB 색상으로 화면 상하단을 채운다.

### 입력 처리와 종료

- 키 입력은 플레이어의 위치 또는 시점 벡터를 갱신하는 이벤트로 처리한다.
- 이동은 벽을 통과하지 않도록 다음 위치를 먼저 검사한 뒤 실제 좌표를 변경해야 한다.
- `ESC`와 창 닫기 이벤트는 할당된 텍스처, 맵, 이미지, 창 자원을 정리하고 종료해야 한다.

## 내가 구현한 방식

### 구조체 역할

- `t_data`: MLX 포인터, window, 정수 맵, 화면 버퍼, 맵 높이, 시작 방향, texture 정보, player vector를 묶어 관리한다.
- `t_tex`: 방향별 텍스처 경로와 픽셀 배열, 텍스처 크기, 천장/바닥 색상, 화면 이미지 정보를 저장한다.
- `t_vec`: 플레이어의 현재 위치, 방향 벡터, camera plane 벡터를 저장한다.
- `t_ray`, `t_rvec`: raycasting 중 사용하는 광선 방향, 맵 좌표, step, side distance, delta distance, 보정 거리 값을 저장한다.
- `t_dfs`: 맵이 외벽으로 닫혀 있는지 검사할 때 방문 배열과 행별 너비를 저장한다.

### 초기화 흐름

```text
main
  -> parser
  -> parse_arg
  -> parse_map
  -> init_vector
  -> init_window
  -> init_texture
  -> mlx_loop
```

- `main`은 `t_data`를 0으로 초기화한 뒤 `parser`로 실행 인자와 `.cub` 파일을 검증한다.
- `parser`는 인자 개수, `.cub` 확장자, 파일 open 여부를 확인한 뒤 설정 정보와 맵을 순서대로 파싱한다.
- 파싱이 끝나면 player vector, MLX window, screen image, texture 배열을 초기화한다.
- `mlx_loop_hook`은 매 loop마다 배경, 벽, window 출력을 다시 수행한다.

### `.cub` 설정 파싱

- `parse_arg`는 `NO`, `SO`, `WE`, `EA`, `F`, `C` 여섯 가지 설정 요소를 읽는다.
- 방향별 텍스처 경로는 `t_tex.path`에 저장하고, 바닥/천장 RGB 값은 `parse_rgb`로 정수 색상 값으로 변환한다.
- `parse_rgb`는 쉼표 개수, 숫자 여부, `0~255` 범위를 검사하고, `R << 16 | G << 8 | B` 형태로 색상을 합친다.
- RGB 값 `0,0,0`과 미입력 상태를 구분하기 위해 저장 시점에 1을 더하고, 최종 검증 후 다시 1을 뺀다.

### 맵 검증과 변환

- `parse_map`은 남은 파일 내용을 `get_next_line`으로 읽어 하나의 문자열로 합친 뒤 개행 기준으로 분리한다.
- 읽은 line 수와 분리된 map height를 비교해 중간에 빈 줄이 섞인 map을 에러로 처리한다.
- `vaild_component`는 공백, `0`, `1`, `N`, `S`, `E`, `W`만 허용하고 시작 방향이 하나만 있는지 확인한다.
- `valid_outerwall`은 DFS 방식으로 열린 공간이나 map 바깥으로 이어지는 경로가 있는지 검사한다.
- 검증이 끝난 map은 가장 긴 행에 맞춰 공백으로 resize한 뒤 정수 배열로 변환한다.
- 정수 map에서는 공백을 `2`, 벽을 `1`, 빈 공간을 `0`, 시작 방향 문자를 `-1`로 저장한다.

### Texture와 Window 준비

- `init_window`는 `mlx_init`, `mlx_new_window`, `mlx_new_image`로 window와 screen image를 만든다.
- screen image는 `mlx_get_data_addr`로 얻은 주소에 매 frame의 픽셀 버퍼를 복사한 뒤 window에 출력한다.
- `init_texture`는 네 방향 텍스처를 XPM 파일에서 읽고, 각 텍스처의 픽셀 값을 `int` 배열로 저장한다.
- 모든 방향 텍스처는 같은 width/height를 가져야 하며, 크기가 다르면 에러로 처리한다.

### Raycasting과 화면 출력

- `draw_background`는 화면 위쪽 절반을 천장 색상, 아래쪽 절반을 바닥 색상으로 채운다.
- `raycasting`은 화면의 모든 x좌표에 대해 camera plane 기준의 광선을 계산한다.
- `calc_dda`는 DDA 방식으로 광선이 다음 x/y 격자 경계 중 어디에 먼저 닿는지 비교하며 벽을 찾는다.
- 벽에 닿으면 보정 거리(`perp_dist`)로 화면에 그릴 벽 높이를 계산하고, 텍스처 y좌표 비율을 정한다.
- `draw_wall`은 충돌 위치와 광선 방향으로 텍스처 x좌표를 구하고, 화면 세로 구간에 맞춰 텍스처 픽셀을 버퍼에 채운다.
- `print_window`는 완성된 버퍼를 screen image에 복사한 뒤 `mlx_put_image_to_window`로 출력한다.

### 이동과 시점 회전

- `press_key`는 `W`, `A`, `S`, `D`, 좌우 방향키, `ESC` 입력을 분기한다.
- 전후 이동은 방향 벡터(`dir_x`, `dir_y`)를 기준으로 하고, 좌우 이동은 plane 벡터(`plane_x`, `plane_y`)를 기준으로 한다.
- 이동 전 다음 좌표가 map에서 벽인지 확인하고, 벽이 아닐 때만 위치를 갱신한다.
- `change_view`는 삼각함수로 방향 벡터와 plane 벡터를 함께 회전시킨다.
- 시작 방향에 따라 회전 부호가 달라지는 문제를 맞추기 위해 특정 시작 방향에서는 회전 속도 부호를 보정한다.

### 에러 처리와 종료

- `err_handler`는 파싱, 메모리, 텍스처, 맵, MLX 초기화 실패 시 `Error` 메시지를 출력하고 할당된 자원을 정리한 뒤 종료한다.
- `exit_process`는 텍스처 배열, 맵, screen image, window를 정리하고 정상 종료한다.
- window 닫기 버튼과 `ESC` 키는 모두 `exit_process` 흐름으로 연결된다.

## 폴더 구조

```text
.
├── includes/
│   ├── cub3d.h
│   ├── define.h
│   ├── key_linux.h
│   └── key_mac.h
├── lib/
│   ├── libft/
│   ├── mlx/
│   └── mlx_linux/
├── maps/
├── srcs/
│   ├── parser*.c
│   ├── raycasting.c
│   ├── draw.c
│   ├── event.c
│   ├── move.c
│   ├── destroy.c
│   └── utils*.c
├── textures/
└── Makefile
```

- `includes`: 공용 구조체, 상수, key code 정의
- `lib/libft`: `get_next_line`과 기본 C 유틸리티
- `lib/mlx`, `lib/mlx_linux`: MiniLibX 라이브러리
- `maps`: 실행 예시로 사용하는 `.cub` 맵 파일
- `srcs/parser*.c`: `.cub` 설정, RGB, map component, 외벽 검증, texture 초기화
- `srcs/raycasting.c`: 광선 계산, DDA, 보정 거리 계산
- `srcs/draw.c`: 배경 색상, 벽 텍스처, screen buffer 출력
- `srcs/event.c`, `srcs/move.c`: 키 입력, 이동, 시점 회전
- `srcs/destroy.c`: 에러 처리, 자원 해제, 정상 종료
- `textures`: XPM 텍스처와 원본 PNG 이미지

## 빌드 및 실행

### 빌드

```bash
make
```

- 결과물은 `cub3D` 실행 파일이다.
- 현재 Makefile은 Linux용 MiniLibX와 X11 링크 옵션을 사용한다.

### 실행

```bash
./cub3D maps/1.cub
```

- `.cub` 맵 파일을 인자로 전달해 실행한다.

### 조작

```text
W / A / S / D : 이동
← / →         : 시점 회전
ESC           : 종료
```
