## 42Seoul

- **42Seoul**은 프랑스에서 시작된 *Ecole 42*를 과학기술정보통신부에서 주관하고, 이노베이션 아카데미에서 운영하는 교육과정입니다.

- _La piscine_ 이라는 한 달간의 테스트 과정을 통과하면 최대 2년간 진행 가능한 본과정을 진행할 수 있으며, 2년안에 본과정을 통과하면 +3년 동안 아웃서클 과제 진행 및 교육장 이용이 가능하다.

- 42seoul 9기

## Norm

- 42에서 과제를 진행하면서 공통적으로 지켜야하는 코드 작성 규칙.
- 과제별 제한사항 및 규칙은 각 과제의 Subject에 명시되어있다.

> [Norm.v3](./ko.norm.pdf)

## 과제 목록

### Circle 0

- **LaPiscine**: 42의 교육 방식과 C/UNIX 기초를 익히는 한 달간의 사전 과정.
  > [README](./00.LaPiscine/README.md)

### Circle 1

- **libft**: C 표준 라이브러리 일부 함수와 리스트 유틸리티를 직접 구현해 정적 라이브러리로 묶는 과제.
  > [README](./01.libft/README.md)
- **get_next_line**: `read`와 정적 변수를 사용해 파일 디스크립터에서 한 줄씩 읽는 함수를 구현하는 과제.
  > [README](./11.get_next_line/README.md)
- **born2beroot**: VM 위에 리눅스 서버를 구성하고 사용자, 권한, SSH, 방화벽, 모니터링을 설정하는 시스템 과제.
  > [README](./12.born2beroot/README.md)
- **ft_printf**: C 표준 함수 `printf`의 일부 서식 기능을 직접 구현하는 가변 인자 출력 과제.
  > [README](./13.ft_printf/README.md)

### Circle 2

- **push_swap**: 두 stack과 제한된 명령어만으로 정렬 명령 목록을 생성하는 알고리즘 과제.
  > [README](./21.push_swap/README.md)
- **so_long**: MiniLibX로 2D 타일 맵 게임을 만들며 맵 파싱, 유효성 검증, 렌더링, 이벤트 처리를 다루는 과제.
  > [README](./22.so_long/README.md)
- **minitalk**: signal만으로 문자열을 bit 단위 전송하는 process 통신 과제.
  > [README](./23.minitalk/README.md)

### Circle 3

- **Minishell**: POSIX 표준 기반 Bash Shell을 모방 구현하는 2인 팀 과제.
  > [README](./31.Minishell/README.md)
- **Philosophers**: 식사하는 철학자 문제를 통해 공유자원을 다루는 동시성 과제.
  > [README](./32.Philosophers/README.md)

### Circle 4

- **Cub3D**: 2D 격자 맵을 raycasting으로 1인칭 3D 화면처럼 렌더링하는 그래픽 과제.
  > [README](./41.Cub3d/README.md)
- **NetPractice**: subnet mask, routing table, gateway를 설정해 제한 시간 안에 네트워크 문제를 해결하는 풀이형 과제.
  > [README](./42.NetPractice/README.md)
- **CPP Module 00**: C++ 기본 문법과 object 중심 구조를 학습하는 첫 번째 C++ 과제.
  > [README](./43-0.CPP-Module_00/README.md)
- **CPP Module 01**: `new/delete`, stack/heap object, pointer/reference, file stream, member function pointer를 다루는 과제.
  > [README](./43-1.CPP-Module_01/README.md)
- **CPP Module 02**: fixed point 표현, Orthodox Canonical Form, operator overloading을 구현하는 과제.
  > [README](./43-2.CPP-Module_02/README.md)
- **CPP Module 03**: inheritance, access modifier, constructor/destructor 호출 순서를 학습하는 과제.
  > [README](./43-3.CPP-Module_03/README.md)
- **CPP Module 04**: polymorphism, virtual function, dynamic binding, deep copy, abstract class를 다루는 과제.
  > [README](./43-4.CPP-Module_04/README.md)

### Circle 5

- **Inception**: Docker Compose로 NGINX, WordPress/PHP-FPM, MariaDB를 분리 구성하고 network, volume, entrypoint, secret 분리를 다루는 인프라 과제.
