# so_long

## 개요

- MiniLibX를 사용해 2D 타일 맵 기반의 간단한 게임을 구현하는 그래픽 과제다.
- `.ber` 맵 파싱, 맵 유효성 검증, 경로 탐색, 키보드 이벤트, 이미지 렌더링, 자원 해제를 함께 다뤘다.
- 플레이어가 모든 수집 요소를 얻은 뒤 출구로 이동하는 흐름을 구현하는 것이 핵심이다.
- 이후 진행하게되는 Cub3D 과제를 위한 예행 연습이라고 볼 수 있다.

## 과제 요구사항 요약

- 맵 파일은 `.ber` 확장자를 사용하고, 직사각형 구조여야 하며 외곽이 벽으로 닫혀 있어야 한다.
- 맵 요소는 벽, 빈 공간, 플레이어 시작 위치, 수집 요소, 출구로 구성되며 개수 조건을 만족해야 한다.
- 플레이어 시작 위치에서 모든 수집 요소와 출구까지 도달 가능한지 확인해야 한다.
- 키 입력으로 이동하고, 벽 충돌을 막고, 이동 횟수를 출력한다.
- MiniLibX로 창을 만들고, 타일/플레이어/수집 요소/출구 이미지를 화면에 그린다.

## 메모

### MiniLibX

- 42에서 제공되는 소형 그래픽 라이브러리로, 창 생성, 이미지 로드, 이미지 출력, 키 입력 이벤트 연결을 직접 다뤄야 한다.
- 운영체제에 따라 의존하는 라이브러리가 달라지는데, 42 클러스터 환경이 macOS이기 때문에 `-framework OpenGL -framework AppKit`로 링크한다.

- 키코드는 환경에 따라 달라질 수 있어, 이 코드는 macOS 기준 방향키와 `W/A/S/D`, ESC 키 값을 상수로 두고 처리한다.
- 키 입력은 `mlx_hook`으로 연결하며, ESC 또는 창 닫기 이벤트가 들어오면 할당한 맵, 이미지, 윈도우 자원을 해제하고 종료한다.

- XPM 이미지는 `mlx_xpm_file_to_image`로 불러오고, 맵의 각 문자(`0`, `1`, `C`, `E`, `P`)에 맞춰 `mlx_put_image_to_window`로 타일 단위 렌더링한다.

### .ber 맵 검증

- `.ber` 파일은 실행 인자로 하나만 받아야 하며, 확장자가 `.ber`인지 먼저 확인한다.
- 파일 내용을 읽어 `\n` 기준으로 2차원 맵 배열을 만들고, 모든 행의 길이가 같은 직사각형 구조인지 검사한다.
- 맵의 외곽은 모두 벽(`1`)이어야 하며, 허용되는 문자는 빈 공간(`0`), 벽(`1`), 수집 요소(`C`), 출구(`E`), 플레이어(`P`)뿐이다.
- 수집 요소는 1개 이상, 출구와 플레이어는 각각 정확히 1개여야 한다.
- 구조 검증이 끝나면 플레이어 시작 위치에서 DFS를 수행해 모든 수집 요소를 지나 출구까지 도달 가능한지 확인한다.
- 플레이어 이동은 좌표 갱신 전에 다음 칸의 벽/출구/수집 요소 상태를 검사하는 방식으로 처리했다.

## 폴더 구조

```text
.
├── maps/
├── mlx/
├── srcs/
├── textures/
└── Makefile
```

- `maps`: 테스트용 `.ber` 맵
- `textures`: 게임에 사용하는 이미지 리소스
- `mlx`: MiniLibX 라이브러리
- `srcs`: 파싱, 검증, 이벤트, 렌더링, 해제 로직

## 빌드 및 실행

### 빌드

```bash
make
```

- 결과물은 `so_long` 실행 파일이다.
- 현재 Makefile은 macOS 프레임워크 기반 MiniLibX 링크 옵션을 사용한다.

### 실행

```bash
./so_long maps/1.ber
```

- 맵 파일 경로를 인자로 전달해 실행한다.
