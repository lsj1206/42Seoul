# push_swap

## 개요

- 두 개의 스택과 제한된 명령어만 사용해 정수 배열을 오름차순으로 정렬하는 알고리즘 과제다.
- 정렬된 배열 자체를 출력하는 것이 아니라, 스택을 정렬하기 위한 명령어 목록을 표준 출력으로 출력한다.

## 과제 요구사항

### Mandatory

- `push_swap` 실행 파일을 생성한다.
- 프로그램은 인자로 받은 정수들을 스택 `a`에 넣고, 스택 `b`는 비어 있는 상태에서 시작한다.
- 첫 번째 인자가 스택 `a`의 top에 위치해야 한다.
- 스택 `a`를 오름차순으로 정렬하기 위한 명령어 목록을 출력한다.
- 명령어는 `sa`, `sb`, `ss`, `pa`, `pb`, `ra`, `rb`, `rr`, `rra`, `rrb`, `rrr`만 사용할 수 있다.
- 각 명령어는 `\n`으로 구분되어야 하며, 그 외의 출력은 없어야 한다.
- 인자가 없으면 아무것도 출력하지 않는다.
- 숫자가 아닌 값, `int` 범위를 벗어나는 값, 중복 값이 들어오면 표준 에러에 `Error\n`를 출력한다.
- 평가에서는 출력한 명령어가 실제로 정렬을 만들 수 있는지와 명령어 개수가 기준 안에 들어오는지를 함께 확인한다.

## 메모

### 입력 검증

- `init_data`에서 여러 인자를 하나의 문자열로 합친 뒤 `ft_split`으로 분리해 처리한다.
- `error_input`은 숫자, 공백, 첫 위치의 `-` 외 문자가 들어오면 에러로 처리한다.
- `ft_atoi` 결과가 `int` 범위를 벗어나면 에러로 처리하고, `error_overlap`에서 중복 값을 검사한다.
- 이미 오름차순으로 정렬된 입력은 명령어를 출력하지 않고 종료한다.

### 스택과 명령어

- 스택 `a`와 `b`는 `top`, `bottom`, `size`를 가진 이중 연결 리스트 형태로 관리한다.
- 사용할 수 있는 명령어는 다음 11개로 제한된다.

| 명령어 | 동작                                                        |
| ------ | ----------------------------------------------------------- |
| `sa`   | `stack_A`의 top 두 원소를 교환                              |
| `sb`   | `stack_B`의 top 두 원소를 교환                              |
| `ss`   | `sa`와 `sb`를 동시에 수행                                   |
| `pa`   | `stack_B`의 top 원소를 `stack_A`의 top으로 이동             |
| `pb`   | `stack_A`의 top 원소를 `stack_B`의 top으로 이동             |
| `ra`   | `stack_A`를 한 칸 위로 회전, 첫 원소는 마지막으로 이동      |
| `rb`   | `stack_B`를 한 칸 위로 회전, 첫 원소는 마지막으로 이동      |
| `rr`   | `ra`와 `rb`를 동시에 수행                                   |
| `rra`  | `stack_A`를 한 칸 아래로 회전, 마지막 원소는 첫 원소로 이동 |
| `rrb`  | `stack_B`를 한 칸 아래로 회전, 마지막 원소는 첫 원소로 이동 |
| `rrr`  | `rra`와 `rrb`를 동시에 수행                                 |

- 명령어는 즉시 출력하지 않고 `print_list`에 저장한 뒤 마지막에 출력한다.
- 출력 직전에 `sa`+`sb`를 `ss`, `ra`+`rb`를 `rr`, `rra`+`rrb`를 `rrr`로 합쳐 명령어 수를 줄인다.

### 정렬 전략

- 일반 정렬처럼 배열을 직접 정렬한 결과를 만들 수 없기 때문에, 현재 값의 상대적 위치를 판단하고 `push`, `rotate`, `reverse rotate` 조합으로 스택 상태를 단계적으로 바꾸는 방식으로 접근한다.
- 비교 횟수보다 출력 명령어 수가 평가에 직접적인 영향을 주므로, 작은 입력 전용 처리와 큰 입력 분할 전략을 분리했다.
- 입력 크기가 2~5개일 때는 `only_two`, `only_three`, `only_four`, `only_five` 전용 케이스로 처리한다.
- 그보다 큰 입력은 `sort_a`와 `sort_b`가 서로 호출되는 재귀 구조로 정렬한다.
- `init_pivot_a`, `init_pivot_b`에서 현재 구간을 배열로 복사하고 정렬해 작은 피벗과 큰 피벗을 구한다.
- `div_atob`는 스택 `a`의 값을 피벗 기준으로 `b`에 나누어 보내고, `div_btoa`는 스택 `b`의 값을 다시 `a`로 보내며 구간을 줄인다.
- `restore`는 분할 과정에서 발생한 회전을 역회전으로 되돌려 다음 재귀 구간의 기준을 맞춘다.

## push_swap 과제의 가치

`push_swap`은 정렬 결과를 직접 만드는 과제가 아니라, 제한된 스택 명령어만으로 정렬 상태에 도달하는 과정을 설계하는 과제다.

일반 정렬 알고리즘의 주요 기준이 시간 복잡도와 비교 횟수라면, `push_swap`의 핵심 기준은 출력 명령어 수다. 그래서 이 과제에서는 값을 빠르게 정렬하는 것보다, 현재 스택 상태를 적은 수의 `push`, `swap`, `rotate` 명령으로 정렬에 가깝게 만드는 전략이 중요하다.

이 과정을 통해 제약 조건에 맞게 문제를 다시 해석하고, 자료구조의 상태 전이를 설계하며, 알고리즘의 비용 기준을 상황에 맞게 바꾸는 경험을 하게 된다.

## 폴더 구조

```text
.
├── header
│   └── push_swap.h
├── srcs
│   ├── main.c
│   ├── push_swap_init.c
│   ├── push_swap_error.c
│   ├── push_swap_utils_*.c
│   ├── command_*.c
│   ├── only_case*.c
│   ├── sort*.c
│   └── printer.c
└── Makefile
```

- `header/push_swap.h`: 스택, 명령어 리스트, 피벗 구조체와 함수 원형
- `srcs/main.c`: 입력 파싱, 정렬 분기, 명령어 출력 흐름
- `srcs/command_*.c`: 스택 명령어 구현
- `srcs/only_case*.c`: 2~5개 입력 전용 정렬
- `srcs/sort*.c`: 피벗 기반 분할 정렬
- `srcs/printer.c`: 명령어 리스트 저장과 출력 최적화

## 빌드 및 실행

### 빌드

```bash
make
```

- 결과물은 `push_swap` 실행 파일이다.

### 실행

```bash
./push_swap 3 2 1
```

- 인자로 받은 정수들을 정렬하기 위한 명령어를 표준 출력으로 출력한다.

### 검증 예시

```bash
ARG="4 67 3 87 23"
./push_swap $ARG | wc -l
./push_swap $ARG | ./checker_OS $ARG
```

- `checker_OS`는 subject에서 제공되는 검증용 바이너리이며, 정렬 성공 시 `OK`를 출력한다.
